# contactbook — Контактная книга

<strong>Волкова Софья</strong><br>
Учебный проект по теме «Использование Intent в Android». Приложение демонстрирует работу с неявными Intent для вызова системных функций: звонки, отправка email, отображение карт и функция «Поделиться».

## Функционал

На главном экране представлены четыре кнопки:

1. **«Позвонить»**  
   Открывает системный номеронабиратель с номером `+7 (495) 123-45-67`.

2. **«Написать email»**  
   апускает почтовый клиент с заполненным адресом `contact@example.com` и темой `Обращение`.

3. **«Показать офис на карте»**  
   Открывает карты с меткой «Наш офис» по координатам `60.0237, 30.2289`.

4. **«Поделиться контактом»**  
   Показывает диалог выбора приложения для отправки текста `Контакт: +7 (495) 123-45-67, contact@example.com`.

Для каждого действия реализована проверка наличия подходящего приложения. Если обработчик не найден, пользователю выводится соответствующее уведомление через `Toast`.

## Особенности реализации

- **Неявные Intent** — для вызова внешних приложений используются только неявные Intent (раздел 2.2).
- **Проверка `resolveActivity()`** — перед запуском каждого Intent выполняется проверка, чтобы избежать краша при отсутствии нужного приложения (раздел 2.3).
- **`Intent.createChooser()`** — для кнопки «Поделиться» используется данный метод, чтобы диалог выбора отображался всегда, даже если у пользователя выбрано приложение по умолчанию (раздел 5.4).
- **`<queries>` в манифесте** — добавлены необходимые элементы для обеспечения видимости почтовых клиентов, звонилок и картографических сервисов в Android 11+ (см. приложенный документ *Настройка AndroidManifest*).
- **Атрибут `android:exported`** — у всех компонентов с Intent Filter явно указано значение `true`/`false` в соответствии с требованиями Android 12+ (раздел 4.5).
- **Схема `mailto:`** — для отправки email используется `ACTION_SENDTO` с URI `mailto:`, что гарантирует показ только почтовых приложений (раздел 5.3).
- **`StrictMode` (в DEBUG-сборке)** — включена проверка небезопасного запуска Intent для обнаружения потенциальных уязвимостей на этапе разработки (раздел 8.2).

## Архитектура и технологии

- Язык: **Kotlin**
- UI: **Jetpack Compose**
- Минимальный SDK: **24** (Android 7.0)
- Целевой SDK: **36**
- Сборка: **Gradle Kotlin DSL**  

## Запуск приложения

1. Клонируйте репозиторий или откройте проект в **Android Studio** (рекомендуется Hedgehog | 2023.1.1 или новее).
2. Дождитесь завершения синхронизации Gradle.
3. Подключите устройство или запустите эмулятор с API 24+.
4. Нажмите **Run** (зелёный треугольник).
